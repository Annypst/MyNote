# JS 定时器 (Timer) 常用方法 + 核心要点 极简总结（笔试 / 面试高频，必背）

JS 的定时器 Timer 是 **window 对象的核心内置方法**，无兼容性问题，专门做「延迟执行 / 定时重复执行」，所有方法都可以省略`window.`直接写，共 **2 个核心创建方法 + 2 个核心清除方法**，是必考考点！

------

## ✅ 一、2 个【创建定时器】的核心方法（最常用）

### 1. 一次性定时器 ：`setTimeout(函数, 延迟毫秒数)`

▸ 作用：**延迟指定时间后，执行【一次】函数代码**，执行完就失效

▸ 语法：`let 定时器名 = setTimeout(回调函数, 毫秒值);`

▸ 示例：延迟 2000ms (2 秒) 后弹出提示，只弹一次



```
let timer1 = setTimeout(function(){
    alert("2秒到了，只执行一次");
}, 2000);
```

### 2. 周期性定时器 ：`setInterval(函数, 间隔毫秒数)`

▸ 作用：**每隔指定时间，重复执行【多次】函数代码**，不手动停止就一直执行

▸ 语法：`let 定时器名 = setInterval(回调函数, 毫秒值);`

▸ 示例：每隔 1000ms (1 秒) 打印一次内容，循环执行





```
let timer2 = setInterval(function(){
    console.log("每隔1秒执行一次");
}, 1000);
```

> ✔ 单位说明：两个方法的时间参数都是 **毫秒 (ms)** ，1 秒 = 1000 毫秒；最小执行间隔约 4ms（浏览器限制）

------

## ✅ 二、2 个【清除定时器】的核心方法（必须配对用）

定时器创建后会返回一个「数字编号」(定时器 ID)，清除时就是通过这个 ID 关闭定时器，**创建和清除必须一一对应，这是笔试必考点**！

### 1. 清除一次性定时器：`clearTimeout(定时器名/ID)`

▸ 作用：关闭 `setTimeout()` 创建的定时器，未执行的代码将不再执行



```
clearTimeout(timer1); // 关闭上面创建的一次性定时器
```

### 2. 清除周期性定时器：`clearInterval(定时器名/ID)`

▸ 作用：关闭 `setInterval()` 创建的定时器，立即停止循环执行，**开发中必写，否则会内存泄漏**



```
clearInterval(timer2); // 关闭上面创建的周期性定时器
```

## ✅ 三、定时器【高频必记考点】（笔试 / 面试挖坑点，全是重点）

### ✔ 考点 1 核心易错：定时器的「执行机制」

定时器是 **异步执行** 的！代码中定时器不会阻塞后续代码执行，会先执行完其他代码，等时间到了再执行定时器内部的函数。





```
console.log(1);
setTimeout(()=>{console.log(2)},0);
console.log(3);
// 执行结果：1 → 3 → 2 （笔试必考输出题！）
```

### ✔ 考点 2 高频写法：定时器的两种函数写法

1. 匿名函数（最常用，一次性定时器）
2. 命名函数（推荐，周期性定时器，便于维护）





```
// 写法1：匿名函数
setTimeout(()=>{alert("匿名函数")},1000);
// 写法2：命名函数（无括号！！笔试大坑）
function fn(){console.log("命名函数")}
setInterval(fn, 1000); // ✔正确 只写函数名，不加()
// setInterval(fn(),1000) ❌错误 加()会立即执行函数，定时器失效
```

### ✔ 考点 3 必用场景：定时器启动 + 停止 组合（开发 / 笔试高频代码题）



```
// 1. 开启定时器，每隔1秒数字+1
let num = 0;
let timer = setInterval(()=>{
    num++;
    console.log(num);
    // 2. 满足条件后，关闭定时器（核心用法）
    if(num >= 5){
        clearInterval(timer);
        console.log("定时器已停止");
    }
},1000);
```

## ✅ 四、极简速记表（全部考点，一行总结，直接背）

|        方法        |            作用            |       配对清除方法       |
| :----------------: | :------------------------: | :----------------------: |
| setTimeout(fn,ms)  | 延迟 ms 毫秒，执行【一次】 | clearTimeout (定时器名)  |
| setInterval(fn,ms) | 间隔 ms 毫秒，执行【多次】 | clearInterval (定时器名) |

> ✔ 终极口诀：**超时只执行一次，间隔循环执行；创建存变量，清除传变量**

------

## ✅ 补充：2 个笔试高频判断题（你 100% 会遇到）

1. 判断：setInterval 创建的定时器，不调用 clearInterval 会一直执行 → ✅ 正确
2. 判断：setTimeout (fn,0) 会立即执行函数 → ❌ 错误（异步机制，等主线程代码执行完再执行）
